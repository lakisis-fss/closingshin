# 03. VCP 시스템 참조 가이드 (VCP System Reference)

이 문서는 Mark Minervini의 **VCP (Volatility Contraction Pattern)** 전략을 ClosingSHIN 시스템에 구현한 핵심 지표, 스코어링 수식, 판정 기준 및 스캐너 사용법을 통합하여 설명합니다.

---

## 1. 핵심 지표 (Core Metrics)

VCP 대시보드와 스캐너가 추적하는 세 가지 주요 지표입니다.

| 지표 | 한글 명칭 | 의미 | 핵심 질문 |
|---|---|---|---|
| **Contraction** | 수축/조정 | 각 파동의 고점-저점 하락폭 이력 (`find_contractions`) | "파동마다 얼마나 떨어졌나?" |
| **Compression** | 압축 | 마지막 수축의 타이트한 깊이 (`last_depth_pct`) | "지금 얼마나 좁게 눌려있나?" |
| **Pivot Distance**| 피벗 거리 | 현재가 - 매수 기준점(최근 고점)까지의 거리 | "매수(Breakout)까지 얼마나 남았나?" |

### 필수 선행 조건: VDU (Volume Dry-Up)
- **정의:** 수축의 마지막 단계에서 거래량이 평소의 50% 이하, 심하게는 10~20%로 급감하는 현상.
- **판단 기준:** 5일 평균 거래량이 50일 평균 거래량의 80% 미만일 때.
- **의미:** 매도 유통 물량이 마른 상태로, 작은 매수세 유입만으로도 폭발적 상승이 가능합니다.

---

## 2. 하이브리드 스코어링 시스템 (Scoring System)

VCP 후보 종목을 '완성도'와 '폭발성' 두 가지 관점에서 평가합니다. (`Scripts/02_scan_vcp.py` 측정)

### 2-1. VCP Score (종합 점수, 0~100점)
가장 안정적이고 교과서적인 VCP 패턴의 완성도를 평가하며, 리스트의 **기본 정렬 기준**이 됩니다.

- **변동성 축소 (40점):** 마지막 조정폭이 5% 미만이면 40점, 10% 미만이면 30점.
- **거래량 고갈 (30점):** VDU 발생 시 +20점, 거래량 비율 극감소 시 추가 +10점.
- **파동 횟수 (30점):** 3~4회(T) 수축 시 가장 이상적(30점), 2회나 5회 이상은 20점.

### 2-2. Jump Score (급등 지수, 0~100점)
스프링이 "얼마나 강하게 눌려있는가?"에 집중하는 단기 폭발성 지수. 80점 이상 시 로켓 아이콘 부여.

- **초압축 변동성 (60점):** 마지막 조정폭이 **3% 미만**으로 극도로 타이트할 경우 60점 부여.
- **파동 건전성 (20점):** 마지막 진동폭이 직전 진동폭보다 작아 점진적 수렴이 일어난 경우.
- **거래량 확인 (20점):** 명확한 VDU 상태인지 확인.

## 3. Action Bar 판정 기준

프론트엔드(`VcpSmartGauge.tsx`)에서 4가지 조건의 충족 여부를 확인하여 매수 준비도(Signal)를 표시합니다.

| 조건 | 라벨 | 충족 기준 |
|---|---|---|
| **Volume Dry Up** | `VDU` | 거래량 고갈 (5일평균 < 50일평균의 80%) |
| **Last Depth** | `< 5%` | 마지막 수축 폭이 5% 미만으로 타이트함 |
| **Pivot Proximity** | `PIVOT` | 현재가가 매수 기준점 대비 `-2% ~ +3%` 이내 |
| **Wave Count** | `3T+` | 수축 파동 횟수가 최소 3회 이상 |

- **4개 충족 (GO):** 파란색 표시. 모든 조건 충족. 브레이크아웃 매수 진입 적극 고려구간.
- **3개 충족 (WATCH):** 노란색 표시. 한 가지 조건이 부족하나 패턴 완성을 눈앞에 둔 주의 구간.
- **0~2개 충족 (WAIT):** 회색 표시. 패턴 미완성으로 대기.

*팁: 치트(Cheat) 타점* - 전체 패턴의 중간 지점에서 나타나는 작은 수축 구간에서 선취매를 시도하는 공격적인 기법도 유효합니다.

## 4. VCP 스캐너 및 시각화 사용법

### 4-1. VCP 스캔 실행
매일 장 마감 후 후보군을 발굴합니다. 스케줄러가 15:40에 자동 실행하며, Admin 페이지에서 수동 트리거도 가능합니다.

```bash
# 수동 실행 (venv 활성화 후)
python Scripts/02_scan_vcp.py
```

- **저장:** PocketBase `vcp_reports` 컬렉션에 날짜별로 기록

### 4-2. VCP 차트 시각화
스캔 결과를 기반으로 캔들, MA(50, 150, 200), 지표 오버레이가 포함된 차트를 생성합니다.

```bash
# 스캔 완료 후 자동 실행 (스케줄러 체이닝)
python Scripts/03_visualize_vcp.py
```

- **저장:** PocketBase `vcp_charts` 컬렉션에 차트 이미지 파일 업로드
