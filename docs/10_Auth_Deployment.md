# ClosingSHIN 대시보드 로그인 기능 배포 가이드 (시놀로지 Docker 환경)

이 문서는 ClosingSHIN 프로젝트에 새롭게 도입된 **프론트엔드 로그인 기능**을 시놀로지(Synology) Docker 환경에서 안전하고 유연하게 관리하기 위한 가이드라인입니다.

## 1. 개요 및 구현 원리
일반적으로 Next.js 프레임워크에서 `NEXT_PUBLIC_` 접두사가 붙은 환경변수는 **빌드(Build) 시점에 코드로 하드코딩**됩니다. 
즉, 이미 만들어진 Docker 이미지 속의 암호를 시놀로지 환경 설정 화면에서 다시 바꾸어 보아도 프론트엔드 화면에는 전혀 반영되지 않는 고질적인 문제가 있습니다.

이를 완벽하게 해결하기 위해 **API Route 우회 방식(Runtime Fetch)**을 도입했습니다.
1. 사용자가 비밀번호를 입력합니다.
2. 프론트엔드는 암호를 브라우저에서 직접 검사하지 않고, Next.js 내부 백엔드 서버(`/api/auth/verify`)로 전송합니다.
3. Next.js 서버는 **시놀로지의 `.env` 파일(또는 Container Manager UI)에 적힌 최신 암호를 실시간으로 읽어서** 맞는지 틀린지 판별만 해줍니다.
4. 암호가 맞다면 30분짜리 브라우저 인가(Session Storage) 타이머가 시작됩니다.

## 2. 시놀로지 적용 방법 (환경변수 설정)
앞으로 Docker 빌드를 새로 하시거나 재시작하실 때, 프론트엔드 전용 시놀로지 환경 설정(.env 또는 UI)에 다음 변수를 추가하시면 됩니다.

```env
SITE_PASSWORD=원하는_비밀번호_입력 (실시간 반영)
```
*(기존 `NEXT_PUBLIC_SITE_PASSWORD`는 더 이상 사용하지 않아도 됩니다)*

### 장점
* Docker 이미지를 딱 1번만 구워(Build)두면, 나중에 비밀번호를 바꾸고 싶을 때는 시놀로지 UI나 `.env` 파일 내용만 수정하고 **컨테이너 재시작만 하면 1초 만에 즉시 반영**됩니다.
* 암호가 자바스크립트 코드 내부에 숨겨지지 않아 해킹이나 소스코드 유출에 안전합니다.
* 30분 간 화면 클릭이나 마우스 이동이 감지되지 않으면, 브라우저 앞단에서 자동으로 로그아웃 시켜 데이터를 보호합니다.
