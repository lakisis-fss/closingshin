# Synology 도커 무한 로딩 시 강제 종료 방법

컨테이너가 연산 부하(과도한 순차적 데이터 요청 등)나 기타 오류로 멈춰버리면, Container Manager(구 Docker) 앱에서도 "로드 중..."이라는 메시지만 뜨고 제어가 전혀 안 되는 먹통 상태(프리징)에 빠질 수 있습니다.

이때는 앱 내부에서 해결하려 하지 말고, **앱 자체의 전원을 껐다 켜는 '강제 종료' 방식**을 써야 합니다. 고등학생도 따라 할 수 있는 3단계 대처법을 안내해 드립니다.

## 1. 1단계: 패키지 센터에서 앱 강제 재시작 (가장 추천)
NAS 전체를 껐다 켜기엔 시간이 너무 오래 걸리므로, 문제가 생긴 'Container Manager' 앱만 강제로 껐다가 켜는 가장 깔끔한 방법입니다.

1. 바탕화면에서 **[패키지 센터]**를 엽니다. (첨부하신 사진 배경에 이미 열려있네요!)
2. 왼쪽 메뉴에서 **[설치됨]** 탭을 클릭합니다.
3. 스크롤을 내려 **[Container Manager]** 앱을 찾아 클릭합니다. (사진에 보이는 팝업창 말고, 패키지 센터 안의 원본 앱 아이콘을 누르셔야 합니다.)
4. 앱 상세 페이지에 들어가면 파란색의 **[열기]** 버튼이 보일 텐데, 그 옆에 있는 작은 **▼ (아래 화살표)** 화살표를 누릅니다.
5. 메뉴에서 **[중지]**를 클릭합니다. (앱이 완전히 멈출 때까지 약 10~30초 정도 기다립니다.)
6. 상태가 '중지됨'으로 바뀌면, 다시 파란색 **[실행]** 버튼을 눌러서 켭니다.

이제 먹통이 풀리고 컨테이너들이 초기화되어 정상 제어가 가능해집니다!

## 2. 2단계: NAS 재부팅 (재시작)
만약 패키지 센터의 [중지] 버튼마저도 빙글빙글 돌면서 먹통이 된다면, NAS 시스템 전체가 꼬인 상태입니다.
- 오른쪽 위 사람 모양(계정) 아이콘 클릭 -> **[다시 시작]**을 눌러 NAS 기기 자체를 깔끔하게 재부팅해 줍니다. 

## 3. 3단계: (전문가용) SSH 터미널 강제 종료
보통 1, 2단계에서 해결되지만, 고급 사용자의 경우 터미널(명령 프롬프트/PuTTY)에 접속해서 강제로 프로세스를 죽이기도 합니다.
- SSH 접속 후 `$ sudo docker kill [컨테이너명]` 명령어 입력
- 도커 엔진 강제 재시작: `$ sudo systemctl restart pkg-ContainerManager-dockerd`
