# 데비안 도커 서버 배포 가이드 (Debian Docker Server Deployment Guide)

## 개요
이 문서는 로컬 PC에서 프로젝트의 프론트엔드(Next.js) 도커 이미지를 생성하고, 이를 WinSCP를 통해 외부 망에 있는 데비안(Debian) 서버로 옮겨 실행하기까지의 전 과정을 정리한 매뉴얼입니다.

현재 네트워크 구성과 목표는 다음과 같습니다.
- **외부 공유기 IP**: `211.224.47.125` (이 주소로 외부에서 접속을 시도하게 됩니다.)
- **데비안 서버(도커 호스트) 내부 IP**: `172.30.1.54`
- **포트 포워딩 규칙**: 외부망의 `3000` 포트로 들어오는 접속을 `172.30.1.54` 데비안 서버의 `3000` 포트로 전달해 줍니다. 

**참고**: 스토리지 데이터베이스인 PocketBase 컨테이너 이미지는 로컬에서 압축해서 따로 가져가지 않아도 됩니다. 데비안 서버에서 컨테이너를 구동할 때 (`docker-compose up`) 인터넷(ghcr.io)을 통해 알아서 다운로드 받기 때문입니다. 이 방식은 우리가 직접 수정한 프로그램(Next.js) 코드만 포장해서 보낼 수 있어 관리하기 매우 효율적입니다.

---

## 1단계: 로컬 PC에서 도커 이미지 생성 및 저장

작성 및 수정한 프론트엔드 코드를 모아, 데비안 리눅스 서버가 이해할 수 있는 배달 상자(도커 이미지)로 만들어 줍니다.

### 1. 프론트엔드 도커 이미지 굽기 (Build)
명령 프롬프트(cmd)나 PowerShell을 열고 현재 프로젝트 폴더로 이동합니다. 그 후 아래 명령어를 입력하여 현재 코드를 엮어 도커 이미지로 구워냅니다 (빌드).

```bash
# 현재 폴더(.)의 파일들을 모아 'closingshin:latest'라는 이름의 도커 이미지로 만듭니다.
docker build -t closingshin:latest .
```
*(참고: `docker-compose build closingshin` 명령어를 사용하셔도 동일하게 내 코드를 이미지 파일로 만들 수 있습니다.)*

### 2. 구워진 이미지를 파일(.tar)로 저장하기
생성이 끝난 이미지를 WinSCP로 서버에 보낼 수 있게끔 하나의 압축 파일로 추출합니다.

```bash
# 만들어진 이미지(closingshin:latest)를 closingshin.tar 라는 실제 파일로 뽑아냅니다.
docker save -o closingshin.tar closingshin:latest
```
이 과정을 거치고 나면 프로젝트 폴더 내부에 용량이 꽤 큰 `closingshin.tar` 파일이 생성됩니다.

## 2단계: WinSCP를 사용하여 데비안 서버로 파일 전송 (업로드)

이제 로컬 윈도우 PC에 포장해둔 파일들을 데비안 서버로 직접 옮겨줄 차례입니다. 파일 원격 전송 프로그램인 WinSCP를 사용합니다.

### 1. WinSCP로 데비안 서버 접속
- **호스트 이름**: 공유기 포트포워딩 설정 혹은 데비안 서버 계정의 IP(공유기 내부 작업 시 `172.30.1.54`)를 적습니다.
- **사용자 이름/비밀번호**: 데비안 서버의 리눅스 계정 정보를 입력합니다.
- **로그인** 버튼을 눌러 접속합니다.

### 2. 서버에 프로젝트 대상 파일 업로드하기
데비안 서버 폴더 안으로 들어와 프로젝트 구동 대상 폴더 (예: `/home/사용자이름/closingshin`) 디렉토리를 하나 만듭니다. 
해당 폴더 안에 다음 파일들을 마우스로 드래그 앤 드롭하여 **업로드** 합니다.

**[반드시 업로드 해야 할 핵심 요소]**
1. **`closingshin.tar`**: 1단계에서 무겁게 포장한 우리의 프론트엔드 압축 파일입니다.
2. **`docker-compose.yml`**: 도커 컨테이너들의 실행 규칙서입니다. 이걸 꼭 올려야 서버가 PocketBase 도 알아서 받아오고 프론트 서버도 켜줄 수 있습니다.
3. **`.env` 파일**: 숨김 파일이라 드래그할 때 빼먹기 쉽습니다! 데이터베이스 접근 정보 등 중요 비밀 정보가 들어있으니 반드시 함께 복사해 주세요.
4. **`pb_data/` 볼륨 폴더**: 로컬에서 임시로 사용 중이던, 혹은 유지해야 할 기존 PocketBase 데이터가 있다면 볼륨 설정을 위해 폴더째 복사해 줍니다.

## 3단계: 데비안 서버에서 도커 컨테이너 실행 (Up)

파일 전송이 완료되었다면 명령어를 칠 수 있는 터미널 프로그램(예: Putty 또는 Windows Terminal)을 켜서 데비안 서버에 SSH로 원격 접속합니다.

### 1. 업로드한 폴더로 들어가 확인하기
데비안 터미널에서 파일을 복사해 둔 폴더로 들어갑니다.

```bash
# 예시: cd /home/사용자이름/closingshin
cd [본인이 파일을 업로드한 서버 내 경로]
```

터미널에 `ls -al` 명령어를 쳐서 `docker-compose.yml`, `.env`, `closingshin.tar` 등의 파일이 폴더 안에 잘 존재하는지 눈으로 확인합니다.

### 2. 구버전 도커 앱 실행 중지 및 제거 (Down)
만약 이미 서버에 이전 버전의 앱이 실행 중이라면, 충돌을 막고 새 버전을 안전하게 구동하기 위해 기존 앱을 내려주어야 합니다.

```bash
# 현재 실행 중인 도커 컨테이너들을 안전하게 종료하고 삭제합니다. (데이터는 지워지지 않습니다)
docker-compose down
```

### 3. 새로운 도커 이미지 로드 (압축 파일 주입)
가짜 파일이 아닌 새로 가져온 프론트엔드용 `.tar` 이미지 파일을 데비안 서버의 도커 시스템 안으로 덮어씌워 줍니다.

```bash
# 복사해 온 프론트엔드 도커 이미지로 기존 버전을 교체(업데이트) 시킵니다.
docker load -i closingshin.tar
```

### 4. 컨테이너 백그라운드 전체 재실행 (Up)
새 이미지를 다 밀어 넣었다면, 다시 `docker-compose.yml` 규칙에 따라 실행 명령을 내립니다! 
서버 도커가 스스로 판단하여 **PocketBase 이미지는 인터넷에서 스마트하게 다운로드 또는 유지** 하고, 두 컨테이너를 동시에 다시 켜줍니다.

```bash
# -d 옵션(Detached Mode)은 터미널 창을 닫아도 컨테이너가 계속 돌아가도록 둡니다.
docker-compose up -d
```

### 5. 정상 작동 확인
컨테이너들이 오류 없이 켜졌는지, 바로 종료되지 않았는지 확인하는 생사 확인 명령어입니다.

```bash
# 현재 실행 중이거나 죽은 도커 컨테이너 상태 전부 확인
docker ps
```

`pocketbase`와 `closingshin` 두 개의 컨테이너가 보이고 상태(STATUS)가 모두 `Up` 이라고 뜬다면, 서버 배포가 완벽하게 마무리된 것입니다.

## 4단계: 접속 확인 및 세팅 완성

서버 상주 준비가 끝났습니다! 이제 바깥에서 스마트폰이나 브라우저 주소창에 다음 주소를 넣어 접속을 시도해 보세요.

- **사이트 접속 주소**: `http://211.224.47.125:3000`

`211.224.47.125`라는 주소를 통해 외부에서 접속 시도를 하면, 외부 IP의 `3000`번 포트로 온 신호를 중간에 있는 공유기가 낚아챕니다. 그리고 공유기에 걸어둔 포트 포워딩 규칙에 의해, 그 신호를 안전하게 데비안 내부망 IP인 `172.30.1.54` 의 3000 포트로 정확히 패스(Pass)해 줍니다. 
그 결과 도커 컨테이너 안에 켜져 있는 우리의 Next.js 서버가 접속한 사람에게 웹 화면을 무사히 띄워주게 됩니다!
