# 다중 VCP 엔진 및 스코어링 시스템 안내

본 문서는 `ClosingSHIN` 프로젝트에 적용된 **다중 VCP(Volatility Contraction Pattern) 스캔 엔진**과 **다형성 스코어링 시스템(Polymorphic Scoring System)** 의 설계 사양 및 동작 원리를 설명합니다.

---

## 1. 다중 VCP 스캔 엔진 개요

기존 단일 형태의 미너비니(Minervini) VCP 스캔 로직을 확장하여, 시장 상황과 사용자의 투자 성향에 맞게 조정할 수 있는 6가지 스캔 모드를 제공합니다. 이를 통해 보수적인 우량주부터 변동성이 큰 야수 테마주까지 폭넓은 탐색이 가능합니다.

### 1.1 스캔 모드별 파라미터 정의

| 모드명 | 설명 | 파라미터 (최소 파동 / 1차 하락 / 마지막 하락) | 추세 요구조건 |
| :--- | :--- | :--- | :--- |
| **Classic (정통)** | 마크 미너비니의 오리지널 VCP 정석 | 2파동 이상 / 35% 이내 / 15% 이내 | 엄격 (정배열) |
| **Aggressive (야수)** | 변동성이 심한 주식, 코인, 단기 테마주 | 2파동 이상 / 60% 이내 / 25% 이내 | 완화 |
| **EarlyBird (얼리버드)** | 횡보 바닥권에서 막 탈출하려는 종목 (Cheat) | 1파동 이상 / 30% 이내 / 15% 이내 | 완화 (50일선) |
| **Stable (대형 우량주)** | 묵직하고 출렁임이 적은 KOSPI 200 등 대형주 | 2파동 이상 / 20% 이내 / 5% 이내 | 엄격 |
| **Relaxed (릴렉스)** | 파동이 줄어들지 않고 박스권을 그리는 종목 | 2파동 이상 / 40% 이내 / 20% 이내 | 보통 |
| **Custom (커스텀)** | 사용자가 패널에서 모든 수치를 직접 입력 | (유저 설정) | (유저 설정) |

### 1.2 엔진 구동 아키텍처
* **Backend (`02_scan_vcp.py`):** `--mode` 인자를 통해 실행 시점에 스캔 모드를 주입받아 동적으로 필터링 변수와 조건을 스위칭합니다. 추출된 데이터프레임에는 `vcp_mode` 메타데이터가 영구 기록됩니다.
* **API (`main.py` & `routes`):** 프론트엔드에서 전달된 JSON 바디의 `mode` 정보 및 `customParams` 객체를 분석하여 파이썬 서브프로세스에 알맞은 파라미터를 넘겨줍니다.

## 2. 다형성 퀀트 스코어링 시스템 (Polymorphic Scoring)

발견된 VCP 후보군에 대해 단순히 '통과/실패' 요소만 제공하는 것을 넘어, **5가지 정량적 지표**를 결합하여 하나의 통합된 점수(Total Quant Score)를 산출합니다.

### 2.1 스코어링 5대 지표
1. **VCP Pattern (패턴 완성도):** 파동의 수, 압축률, 마지막 깊이, 거래량 마름(Volume Dry Up) 조건의 완벽함을 점수화.
2. **Supply / Demand (수급):** 기관/외국인의 순매수 트렌드, 거래대금 폭발(Pocket Pivot) 여부.
3. **AI Sentiment (AI 센티먼트):** 최신 뉴스 기사의 긍정/부정 비율 및 VCP 돌파 모멘텀 분석 점수.
4. **Fundamentals (펀더멘탈):** 어닝 서프라이즈, 영업이익률 현황 등 기업 가치 스코어.
5. **Sector / RS (섹터 상대강도):** 시장(KOSPI/KOSDAQ) 대비 해당 종목 및 속한 섹터의 강도(Relative Strength).

### 2.2 동적 가중치 (Dynamic Weights)
사용자는 프론트엔드의 **[WEIGHTS] (뷰 셋팅)** 패널을 통해 5대 지표가 총점에 미치는 **반영 비율(가중치)**을 실시간으로 조절할 수 있습니다. 

* **기본 모드 (Balanced):** `VCP(30%)`, `수급(20%)`, `AI센티먼트(20%)`, `펀더멘탈(15%)`, `섹터(15%)`
* **차티스트 모드:** `VCP(60%)`, `수급(30%)`, `나머지(10%)`
* 사용자가 가중치를 변경하는 즉시(On-the-fly) 프론트엔드 상태 머신이 작동하여 전체 후보군의 퀀트 스코어를 다시 계산하고, 선택된 정렬 기준(`SORT BY`)에 맞춰 리스트를 재배열합니다.

## 3. UI/UX 통합 가이드 (프론트엔드)

본 구조는 Bauhaus / Brutalism 디자인 감각을 극대화하여 구성되었습니다.

* **Scan Engine & Data Pod:** 하단 대시보드에서 `SCAN ENGINE & DATA` 영역을 통해 날짜를 선택하고, 드롭다운에서 스캔 모드를 선택한 후 **START VCP SCAN** 을 누르면 백엔드 스크립트가 구동됩니다.
* **View Settings Pod:** 우측 혹은 독립된 모듈로 배치된 `VIEW SETTINGS` 패널에서는 동적 스코어링의 가중치(`WEIGHTS`)와 정렬 기준(`SORT BY`)을 튜닝합니다.
* **결과 카드 시각화:** 각 종목 카드 안테나는 산출된 Total Score(0~100점) 배지를 크게 표시하며, 세부 VCP Metrics 패널에는 각 지표별 점수와 비중을 나타내는 컬러 블록(Gauge)이 랜더링됩니다.

---

## 4. 향후 활용 방안 및 확장 (Backtesting)

포트폴리오에 진입하는 모든 종목은 **어떤 스캔 모드(`vcp_mode`)로 발굴되었고, 당시의 합산 스코어가 몇 점이었는가**를 영구적으로 저장합니다. 
이를 기반으로 향후 백테스트 시스템과 연동해, "어떤 모드와 세팅이 현 장세에서 수익률 1위인가"에 대한 딥러닝 피드백 루프를 구축할 수 있습니다.
