# 포트폴리오 수동 청산 (Manual Position Close)

## 📌 배경 및 목적 (Background & Objective)
현재 시스템은 설정된 익절/손절 조건(Take Profit / Stop Loss / Time Cut)에 따라 자동으로 청산 여부를 계산하는 로직은 갖추어져 있으나, 사용자가 실시간으로 현재의 수익률(예: 한화갤러리아 53.93%)에 만족하여 **수동으로 즉시 청산**을 원할 때 이를 시스템에 기록하는 통로가 없습니다.
수동으로 청산하고 해당 내용을 영구적으로 보존 및 관리할 수 있는 기능이 필수적입니다.

---

## 💡 구현 아이디어 (Implementation Ideas)

### 1단계: UI (버튼 제공)
- 포트폴리오 대시보드 화면(또는 상세보기 패널)에서 상태가 `OPEN`인 종목에 한하여 **[수동 청산] (또는 [Close Position])** 버튼을 노출합니다.

### 2단계: 모달 팝업 (Confirmation Modal)
사용자가 실수로 버튼을 눌러 청산되는 것을 방지하고, 명확한 데이터를 남기기 위해 팝업창(모달)을 띄웁니다.

- **표시 정보:**
  - 종목명 및 티커
  - 보유 수량 및 진입가
  - 시스템이 조회한 **현재가** 및 **실시간 예상 수익률(PnL)**
- **입력 가능 정보 (Option):**
  - **청산 가격 수정창:** 기본값은 '현재가'로 채워지나, 사용자가 방금 실제 HTS에서 체결시킨 정확한 가격으로 수정할 수 있도록 허용합니다.
  - **청산 메모 (사유):** "수익률 50% 달성으로 1차 전량 매도"와 같이 차후 복기를 위한 메모를 남길 수 있게 합니다.

### 3단계: 데이터베이스 업데이트 (`portfolio.json`)
확인 버튼을 누르면 API를 통해 해당 종목의 `simulation` 객체가 다음과 같이 영구 변경됩니다.

```json
"simulation": {
  "enabled": true,
  "status": "CLOSED",
  "exitDate": "2026-02-25T12:47:00.000Z", // 수동 청산 버튼을 누른(체결된) 시간
  "exitPrice": 3425, // 현재가 또는 사용자가 입력한 체결가
  "exitReason": "MANUAL_CLOSE (메모: 수익 만족)",
  "exitYield": 53.93,
  "realizedPnl": 1500000,
  "lastUpdate": "2026-02-25T12:47:00.000Z"
}
```

---

## 🛠️ 기대 효과 (Expected Benefits)
- **정확한 수익률 관리:** 시스템 자동 청산과 실제 사용자 수동 청산이 분리되어 정확한 트레이딩 복기가 가능합니다.
- **파이썬 스크립트 대응:** 이전 작업에서 구현한 파이썬의 과거 `OHLC` 백테스팅 로직은 이미 `status: "CLOSED"`인 종목을 무시(Skip)하도록 짜여 있으므로, 수동 청산 기능을 추가해도 모니터링 로봇 및 가치 평가 엔진과 100% 깔끔하게 호환됩니다.
