# 시놀로지 배포 환경 브라우저 에러 분석 및 해결 가이드

## 1. 문제 현상
시놀로지 Docker 환경에서 대시보드 로그인 후 "Application error: a client-side exception has occurred" 메시지가 표시되며 화면이 멈추는 현상입니다.

## 2. 원인 분석 (왜 이런 일이 생기나요?)
이 에러는 자바스크립트가 존재하지 않는 값(undefined 또는 null)을 마치 숫자인 것처럼 처리하려고 할 때 발생합니다.

예를 들어 컴퓨터에게 다음과 같은 명령을 내린 것과 같습니다:
* 사과가 0개인데 "그 사과를 3등분해서 소수점 둘째 자리까지 보여줘"라고 말함.
* 컴퓨터는 존재하지 않는 사과를 자를 수 없으므로 작업을 포기하고 에러를 냅니다.

시놀로지 환경에서는 실시간 데이터 수집이 지연되거나 파일 읽기 권한 문제로 인해 특정 데이터(예: 국제유가, 국채 금리 등)가 비어 있는 경우가 많습니다. 이때 코드에서 .toFixed()(소수점 고정)나 .toLocaleString()(쉼표 표시) 같은 함수를 강제로 호출하면 브라우저가 멈추게 됩니다.

## 3. 해결 방안 (어떻게 고치나요?)
모든 데이터 접근 시 "값이 없을 수도 있다"는 가정을 하고 안전 장치를 추가합니다 (방어적 코딩).

1. 기본값 설정: 데이터가 없으면 강제로 0을 사용하도록 합니다. (|| 0)
2. 조건부 렌더링: 데이터가 확실히 존재할 때만 함수를 호출합니다.
3. 타입 안전성 확보: 숫자 데이터가 확실한지 한 번 더 확인합니다.

## 4. 상세 수정 내역

### 4.1. 대시보드 메인 화면 (page.tsx)
* WTI_OIL, US10Y, SOX 등의 데이터에 접근할 때 연쇄적인 물음표 연산자(?.)와 기본값 처리를 적용합니다.

### 4.2. 마켓 상태 바 (MarketStatusBar.tsx)
* 환율 및 나스닥 데이터가 로딩 중이거나 누락되었을 때 텍스트가 깨지지 않도록 예외 처리를 추가합니다.

### 4.3. 인증 보호 컴포넌트 (PasswordProtection.tsx)
* 인증 성공 후 화면을 전환할 때 브라우저의 저장된 상태와 실제 코드의 상태가 충돌하지 않도록 보정합니다.

## 5. 기대 효과
이 조치를 완료하면 데이터 수집이 덜 된 상태에서도 "데이터 없음"이라는 표시가 뜰 뿐, 페이지 전체가 멈추는 현상은 완벽히 사라집니다.
