# 주식 데이터 분석 대시보드 UI/UX 구현 계획서 v1.0

이 문서는 ClosingSHIN 프로젝트의 데이터를 사용자가 쉽고 직관적으로 볼 수 있도록 돕는 웹 대시보드의 화면 설계 및 구현 계획입니다. **"고등학생도 이해할 수 있는 쉬운 투자 비서"**를 목표로 합니다.

---

## 1. 디자인 철학 (Design Philosophy)

1.  **직관성 (Intuitive):** 복잡한 숫자는 숨기고, **색상(신호등)**과 **아이콘**으로 먼저 보여줍니다.
    *   예: "나스닥이 -2.5% 하락했습니다" 대신 → ☔ **[약세장 경보]** 아이콘 노출.
2.  **스토리텔링 (Storytelling):** 데이터를 나열하지 않고, **흐름(Flow)**을 만들어줍니다.
    *   시장 날씨 확인 -> 쇼핑 리스트(VCP) 확인 -> 물건 꼼꼼히보기(상세 분석) -> 구매 결정.
3.  **결론 중심 (Action-Oriented):** "그래서 사요, 마요?"에 대한 힌트를 줍니다.
    *   단순 차트보다 **AI의 한 줄 코멘트**를 최상단에 배치합니다.

---

## 2. 사이트 구조 (Site Map)

웹사이트는 크게 3가지 핵심 페이지로 구성됩니다.

1.  **🏠 메인 대시보드 (Dashboard):** 시장의 날씨(Market Status)를 한눈에 확인.
2.  **📋 VCP 유망 종목 (Candidates):** 살만한 종목 리스트 (Shopping List).
3.  **🔍 종목 상세 리포트 (Stock Detail):** 차트, 뉴스 AI 분석, 수급, 펀더멘털 통합 뷰.

---

## 3. 페이지별 상세 설계 (Wireframe & Features)

### 3.1 🏠 메인 대시보드 (오늘의 증시 날씨)

사용자가 가장 먼저 보는 화면입니다. 오늘 주식을 사도 되는 날인지 아닌지 3초 만에 판단하게 해줍니다.

**[핵심 컴포넌트]**
-   **헤더:** "오늘의 투자 날씨: **맑음 ☀️**" (종합 점수 기반)
-   **신호등 카드 (Traffic Lights):**
    -   🟢 **환율:** 1,320원 (안정)
    -   🔴 **외국인 수급:** -2,000억 (매도세)
    -   🟡 **ADR:** 95% (보통)
-   **글로벌 체크:** 미국 국채 금리, 유가, 나스닥 지수 (간략한 스파크라인 차트)
-   **섹터 지도:** 오늘 가장 뜨거운 섹터 Top 3 (예: 🔥 반도체 +3.5%)

**[사용자 시나리오]**
> "오늘 달러가 좀 튀었네? 그래도 반도체 쪽에 외국인 매수가 들어오니까, 반도체 VCP 종목만 좀 볼까?"

### 3.2 📋 VCP 유망 종목 (오늘의 쇼핑 리스트)

시장이 괜찮다면, 어떤 종목을 살지 고르는 메뉴판입니다.

**[리스트 구성]**
-   **카드형 UI:** 엑셀 표 대신 카드 형태로 종목 정보를 보여줍니다.
-   **뱃지(Badge) 시스템:**
    -   🏷️ `3 Tight` (3번 웅크림, 매우 좋음)
    -   💧 `Volume Dry` (거래량 말라버림, 폭발 임박)
    -   🤖 `AI Pick` (뉴스 호재 강력)
-   **정렬 필터:** `변동폭 좁은 순`, `AI 점수 높은 순`, `기관 매수 많은 순`

**[사용자 시나리오]**
> "오, 삼성전자가 리스트에 떴네? '3 Tight' 뱃지도 있고.. AI가 뭐라고 하는지 눌러보자."

### 3.3 🔍 종목 상세 리포트 (꼼꼼히 뜯어보기)

종목 카드를 클릭하면 나오는 상세 페이지입니다. 모든 판단 근거를 모아놓습니다.

**[레이아웃 등]**
-   **Zone 1: 차트 (Visual)**
    -   고해상도 캔들 차트 이미지 (이평선, 거래량 포함).
    -   차트 위에 VCP 패턴 구간을 박스로 표시해주면 베스트.
-   **Zone 2: AI 뉴스 브리핑 (Reason)**
    -   **"3줄 요약":** 최근 뉴스들의 핵심 내용.
    -   **"호재 vs 악재":** 게이지 바(Gauge Bar)로 긍정/부정 비율 표시.
    -   **"지속성":** 이 재료가 단타용인지 장투용인지 표시.
-   **Zone 3: 수급과 기본기 (Data)**
    -   **수급 히트맵:** 5일/15일/30일 누가 샀나? (빨강=매수, 파랑=매도)
    -   **펀더멘털:** PER, PBR이 업종 평균보다 싼가 비싼가?

## 4. 단계별 구현 로드맵 (Roadmap)

고등학생 친구와 함께 만든다고 가정하고, 쉬운 것부터 차근차근 만듭니다.

### 1단계: 뼈대 만들기 (MVP - Minimum Viable Product)
*   **목표:** 일단 데이터가 화면에 나오게 하자. 디자인은 나중에.
*   **작업:**
    *   Python 스크립트로 만든 JSON/CSV 파일을 읽어서 그냥 표(Table)로 보여준다.
    *   차트 이미지는 클릭하면 팝업으로 뜨게 한다.
    *   (소요시간: 약 1~2일)

### 2단계: 예쁘게 꾸미기 (UI Polish)
*   **목표:** 보고 싶게 만들자.
*   **작업:**
    *   Tailwind CSS를 써서 카드 디자인 적용.
    *   '매수 신호', '경고' 같은 뱃지에 알록달록 색깔 입히기.
    *   반응형 웹 (모바일에서도 보이게).
    *   (소요시간: 약 2~3일)

### 3단계: 똑똑하게 만들기 (Interaction)
*   **목표:** 사용 편의성 강화.
*   **작업:**
    *   종목 리스트 검색/필터 기능 (예: "반도체만 보여줘").
    *   차트 위 마우스 올리면 주가 나오게 하기 (Interactive Chart).
    *   (소요시간: 약 3~4일)

---

## 5. 기술 스택 추천 (Tech Stack)

*   **프레임워크:** **Next.js** (React 기반, 데이터 보여주기에 최적)
*   **스타일링:** **Tailwind CSS** (빠르고 예쁜 디자인)
*   **차트 라이브러리:** **Recharts** 또는 **Chart.js** (쉬운 차트 구현)
*   **아이콘:** **Lucide React** (깔끔한 아이콘 무료 사용)

---

## 6. 결론

이 UI/UX 계획의 핵심은 **"겁주지 않기"**입니다. 복잡한 수치 대신 직관적인 신호와 요약을 제공하여, 초보 투자자도 전문가처럼 의사결정을 내릴 수 있도록 돕는 것이 최종 목표입니다.
