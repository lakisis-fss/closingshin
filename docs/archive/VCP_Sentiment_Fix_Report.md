# VCP Sentiment 데이터 누락 원인 분석 및 복구 보고서

## 1. 개요
3월 27일 수행된 VCP 스캔 결과에서 모든 종목의 AI Sentiment 점수가 50점으로 표시되고, 뉴스 데이터가 조회되지 않는 현상이 발생했습니다. 이에 대한 원인을 분석하고 복구 방안을 정리합니다.

## 2. 현상 분석
- **증상**: 대시보드에서 AI Sentiment 점수가 50점으로 고정됨. "해당 기간 뉴스 없음" 메시지 출력.
- **데이터 확인**: `vcp_reports` 데이터는 정상적으로 존재하나, 이를 뒷받침하는 `news_analysis` 레코드가 없거나 프런트엔드에서 매칭되지 않음.

## 3. 원인 분석
### 3.1 PocketBase 서버 다운
가장 결정적인 원인은 **PocketBase 서버(`pocketbase.exe`)의 비정상 종료**였습니다. 
- VCP 스캔 프로세스는 종목 선별(`02_scan_vcp.py`) 후 뉴스 수집(`04_collect_news.py`) 및 분석(`05_analyze_news.py`) 스크립트를 차례로 호출합니다.
- 서버가 다운된 상태에서 스캔이 시작되어 종목 데이터는 어떠한 이유로(이전 실행 결과 혹은 부분 성공) 저장되었을 수 있으나, 후속 뉴스 수집/분석 과정에서 DB 연결 실패로 인해 데이터가 저장되지 못했습니다.

### 3.2 뉴스 수집 및 분석 지연
- 수동 스캔 시 뉴스 수집과 분석은 백그라운드에서 진행됩니다. 모든 분석이 완료되기 전에 대시보드를 확인하면 점수가 아직 계산되지 않아 기본값인 50점으로 표시될 수 있습니다.

## 4. 해결 및 복구 방안
- **서버 재기동**: 현재 PocketBase 서버를 다시 가동하여 정상화했습니다.
- **데이터 재수집**: 3월 27일자 날짜를 지정하여 뉴스 수집 및 분석 스크립트를 수동으로 재실행합니다.
- **예방 조치**: 스케줄러 및 자동 시작 스크립트에 PocketBase의 상태를 체크하고 자동 재시작하는 로직을 강화할 필요가 있습니다.

## 5. 단계별 실행 계획
1. `pocketbase.exe` 상태 상시 모니터링
2. `python Scripts/04_collect_news.py --date 20260327` 실행 (뉴스 수집)
3. `python Scripts/05_analyze_news.py --date 20260327` 실행 (AI 분석)
4. 대시보드 데이터 갱신 확인
