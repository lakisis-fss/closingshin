# Git 특정 커밋으로 되돌리는 방법 가이드

Git에서 과거로 돌아가는 방법은 상황에 따라 'Reset'과 'Revert' 중 하나를 선택해야 합니다.

## 1. 커밋 기록 확인 (ID 찾기)
원하는 시점으로 돌아가기 위해 먼저 커밋 해시(ID)를 확인합니다.
```bash
git log --oneline
```
예시 결과:
`a1b2c3d` (최신 - 현재 위치)
`e5f6g7h` (되돌리고 싶은 시점)

---

## 2. Git Reset (역사 재작성)
타임머신을 타고 과거로 여행하는 것과 같습니다. 특정 시점 이후의 커미 기록을 삭제합니다.

### 2.1 종류별 옵션
1. **--hard (가장 강력함)**
   - 지정한 커밋 이후의 모든 코드 변경사항을 **완전히 삭제**합니다.
   - 주의: 복구가 매우 어려우므로 신중해야 합니다.
   ```bash
   git reset --hard [커밋ID]
   ```
2. **--soft**
   - 커밋 기록만 되돌리고, 파일의 수정 내용은 그대로 둔 채 **Staging Area(커밋 대기 상태)**에 남겨둡니다.
   - 코드를 다시 수정해서 새로 커밋하고 싶을 때 유용합니다.
   ```bash
   git reset --soft [커밋ID]
   ```
3. **--mixed (기본값)**
   - 커밋 기록을 되돌리고 파일을 보존하지만, Staging Area에서는 내립니다. (Unstaged 상태)

---

## 3. Git Revert (안전한 취소)
과거의 잘못을 취소하는 **'반성문 커밋'**을 새로 생성합니다. 

- **특징**: 기존의 커밋 역사를 삭제하지 않으므로 팀원들과 공유된 저장소에서 사용할 때 안전합니다.
- **사용법**:
  ```bash
  git revert [취소하고 싶은 커밋ID]
  ```
  - 아이디를 입력하면 해당 커밋만 취소하는 새로운 커밋이 생성됩니다.

---

## 4. 원격 저장소(GitHub) 반영 시 주의사항
`git reset --hard`를 사용했다면 로컬 역사가 원격과 달라져 일반적인 `git push`가 거부됩니다. 이때는 강제 푸시가 필요합니다.
```bash
git push origin [브랜치명] --force
```
**주의**: 팀 프로젝트에서 `--force`를 사용하면 다른 팀원의 로컬 저장소가 꼬일 수 있으므로 반드시 상의 후 사용해야 합니다.
