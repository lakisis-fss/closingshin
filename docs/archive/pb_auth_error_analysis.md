# [ScriptRunner] PocketBase 인증 자동 취소 에러 분석 보고서

## 1. 에러 현상
`npm run scheduler` 실행 터미널에서 다음과 같은 에러 메시지가 반복적으로 발생함:
- `PB Auth Failed: ClientResponseError 0: The request was aborted (most likely autocancelled)`
- `Failed to log to PB: ClientResponseError 0: The request was aborted`

## 2. 원인 분석
### 핵심 원인: PocketBase SDK의 자동 취소(Auto-cancellation) 기능
PocketBase의 자바스크립트 SDK는 성능 최적화와 서버 부하 방지를 위해 **"동일한 요청이 완료되기 전에 다시 들어오면 이전 요청을 자동으로 취소"**하는 기능이 기본적으로 활성화되어 있습니다.

### 상세 발생 과정
1.  스케줄러가 파이썬 스크립트를 실행하고, 스크립트가 여러 줄의 결과(stdout)를 아주 빠른 속도로 출력합니다.
2.  `scriptRunner.ts`는 매 줄마다 로그를 남기기 위해 `appendLog` -> `initPB`를 호출합니다.
3.  아직 첫 번째 인증(`authWithPassword`)이 완료되지 않았는데, 다음 줄의 로그 때문에 두 번째 인증 요청이 발생합니다.
4.  SDK는 "어? 똑같은 인증 요청이 또 들어왔네? 이전 건 취소하자!"라며 첫 번째 요청을 강제로 중단(Abort)시킵니다.
5.  이로 인해 `AbortError`가 발생하고 로그 기록이 실패하게 됩니다.

## 3. 해결 방안 (구현 계획)
1.  **자동 취소 비활성화**: 서버 환경에서는 요청이 취소될 필요가 없으므로 `pb.autoCancellation(false)` 설정을 추가합니다.
2.  **인증 요청 통합 (싱글톤)**: 인증이 진행 중일 때는 새로운 요청을 보내지 않고 기시작된 인증이 끝날 때까지 기다리도록 로직을 수정합니다. (마치 화장실에 사람이 있으면 밖에서 기다리는 것과 같습니다.)

## 4. 예시를 통한 설명
로그인 버튼을 매우 빠르게 10번 광클하는 상황을 상상해 보세요. 
- **이전 상황**: 시스템이 1~9번째 클릭을 모두 "취소된 요청"으로 처리하고 에러를 내뱉는 상황입니다.
- **해결 후**: 첫 번째 클릭 때 시스템이 "지금 로그인 중이니 잠시만 기다려주세요"라고 안내하고, 2~10번째 클릭은 첫 번째 로그인이 끝날 때까지 조용히 대기했다가 로그인된 상태를 함께 이용하게 됩니다.
