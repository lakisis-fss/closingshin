# 개발용 PocketBase 서버 구동 가이드

이 문서는 개발 과정에서 Docker 없이 PocketBase를 빠르고 간편하게 실행하는 방법을 안내합니다.

## 1. 왜 Docker 없이 실행하나요? (개발 효율성)

- **빠른 시동**: 컨테이너를 올리고 내리는 대기 시간 없이 즉시 실행됩니다.
- **간편한 디버깅**: 터미널에서 서버의 실시간 로그를 바로 확인할 수 있습니다.
- **리소스 절약**: Docker Desktop을 띄우지 않아도 되므로 컴퓨터 자원을 적게 사용합니다.

## 2. 서버 실행 방법

윈도우 터미널(PowerShell 또는 CMD)에서 프로젝트 루트 폴더로 이동한 뒤 다음 명령어를 입력하세요.

```powershell
.\pocketbase.exe serve
```

---

## 3. 접속 주소

서버가 실행되면 다음 주소들을 통해 데이터베이스와 관리 도구에 접속할 수 있습니다.

- **API 주소**: `http://127.0.0.1:8090` (프론트엔드와 백엔드가 데이터를 주고받는 통로)
- **관리자 대시보드**: [http://127.0.0.1:8090/_/](http://127.0.0.1:8090/_/) (데이터를 직접 눈으로 확인하고 수정하는 곳)

---

## 4. 주의사항

- **터미널 유지**: 서버를 실행한 터미널 창을 끄면 데이터베이스 서버도 함께 종료됩니다. 개발 중에는 항상 켜두세요.
- **포트 충돌**: 만약 8090 포트가 이미 사용 중이라는 에러가 나오면, 다른 프로그램이 해당 포트를 쓰고 있는 것이니 확인이 필요합니다.

---
> [!TIP]
> **개발 흐름 추천**: 
> 1. `pocketbase.exe serve` 실행 (터미널 1)
> 2. `npm run dev` 프론트엔드 실행 (터미널 2)
> 3. 필요한 파이썬 스크립트 실행 (터미널 3)
