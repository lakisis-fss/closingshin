# 실시간 시장 데이터 반영을 위한 수집 엔진 고도화 계획

## 1. 개요
현재 `05_collect_market_status.py`가 Yahoo Finance의 과거 일봉 데이터를 조회하면서 장 중(Intra-day) 실시간 가격을 반영하지 못하는 문제를 해결합니다.

## 2. 변경 사항 상세요약

### A. 시장 운영 시간 판단 로직 추가
- 한국 거래소(KRX) 운영 시간(09:00 ~ 15:30) 및 데이터 정리 시간(15:40까지)을 고려하여 현재가 수집 여부를 결정합니다.

### B. `_fetch_kr_index` 함수 수정
- `fdr.DataReader`로 가져온 데이터의 마지막 날짜가 오늘(TODAY)이 아닐 경우, 네이버 금융에서 실시간 지수(KOSPI, KOSDAQ)를 크롤링하여 데이터를 보정합니다.

### C. `get_sector_etfs` 함수 수정
- 섹터 ETF(8종)에 대해서도 동일하게 실시간 가격 반영 로직을 추가합니다.

### D. `get_macro_indicators` 함수 수정
- USD/KRW(환율), US10Y(미 국채), WTI(유가) 등의 매크로 지표도 실시간 소스를 우선하도록 변경합니다.

## 3. 작업 파일
- `Scripts/05_collect_market_status.py`

## 4. 검증 계획
1. 수정된 스크립트를 수동 실행하여 `status_YYYYMMDD.json`에 오늘 날짜의 데이터와 현재가가 올바르게 저장되는지 확인.
2. 프론트엔드 대시보드에서 KOSPI/KOSDAQ 값이 실제 네이버 금융 수치와 일치하는지 대조.
