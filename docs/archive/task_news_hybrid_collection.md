# 뉴스 수집 시스템 고도화: 하이브리드(API+크롤링) 방식 도입

## 1. 개요
인기 종목의 뉴스 방출량이 네이버 API의 수집 한계(1000건)를 초과하여 과거 특정 시점의 데이터가 누락되는 문제를 해결하기 위해, API 수집과 웹 크롤링을 결합한 하이브리드 시스템을 구축하였습니다.

## 2. 주요 변경 사항
- 네이버 API 사용 시 1000건 한계 도달 여부를 감지하는 로직 추가
- API 한계 도달 시 또는 데이터 부족 시 자동으로 웹 검색 크롤링 모드로 전환
- 네이버의 최신 Fender UI(Fender UI) 구조에 최적화된 데이터 추출 셀렉터 적용
- User-Agent 순환 및 랜덤 지연 시간을 통한 보안 강화

## 3. 기술적 세부 사항
- 트리거 조건: API 1000번째 뉴스 날짜가 타겟 날짜보다 최신일 경우
- 크롤링 전략: ds(시작일), de(종료일) 파라미터를 사용한 특정 날짜 정밀 타겟팅
- 데이터 병합: 기존 API 데이터와 크롤링 데이터의 중복을 제거하고 단일 CSV로 저장

## 4. 기대 효과
- 한화에어로스페이스, 삼성전자 등 대형주의 과거 데이터 수집 안정성 확보
- 뉴스 데이터 누락으로 인한 수급 및 심리 점수 0점 오류 해결
- API 사용량을 최적화하면서도 데이터의 완전성 보장
