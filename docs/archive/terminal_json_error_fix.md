# 터미널 JSON 파싱 에러 해결 보고서

터미널에서 발생하던 `SyntaxError: Unexpected end of JSON input` 에러를 해결한 내역을 정리한 문서입니다.

## 1. 에러 현상과 원인

### 현상
터미널 창에 붉은색 글씨로 `Failed to fetch progress: SyntaxError: Unexpected end of JSON input`이라는 메시지가 계속 출력되었습니다.

### 원인
이 에러는 마치 책의 마지막 페이지가 찢겨 나간 것처럼, 컴퓨터가 읽으려던 데이터가 도중에 끊겼을 때 발생합니다.
구체적으로는 다음과 같은 상황에서 발생했습니다.
- 배경에서 실행 중인 파이썬 프로그램이 진행 상황(`scan_progress.json`)을 업데이트할 때, 아주 짧은 순간 동안 파일의 내용을 지우고 새로 씁니다.
- 그 찰나의 순간에 웹 서버(Node.js)가 해당 파일을 읽으려고 시도하면, 파일이 텅 비어있거나 데이터가 완성되지 않은 상태로 읽히게 됩니다.
- 텅 빈 데이터를 JSON 형식으로 해석(`JSON.parse`)하려다 보니 "데이터의 끝이 예상치 못하게 끊겼다"는 에러가 발생한 것입니다.

## 2. 해결 방법

문제를 해결하기 위해 두 가지 안전 장치를 추가했습니다.

### 첫 번째 안전 장치: 빈 파일 확인
파일을 읽은 직후, 내용이 비어 있는지 먼저 확인합니다. 만약 비어 있다면 "데이터를 기다리는 중(Waiting for progress data...)"이라는 메시지와 함께 기본 값을 반환하도록 수정했습니다. 이렇게 하면 컴퓨터가 억지로 빈 데이터를 해석하려고 시도하지 않습니다.

### 두 번째 안전 장치: 파싱 예외 처리 (Try-Catch)
데이터가 비어 있지는 않지만, 아직 작성이 덜 끝나서 형식이 맞지 않을 경우를 대비했습니다. 데이터를 해석하는 부분을 '보호 구역(try-catch)'으로 감싸서, 만약 해석 중에 에러가 나더라도 시스템이 멈추지 않고 "데이터 동기화 중(Synchronizing progress data...)"이라는 상태를 반환하며 다음 기회를 기다리게 했습니다.

## 3. 수정된 파일 경로
- `src/app/api/system/scan-progress/route.ts`

## 4. 기대 효과
- 터미널에 더 이상 붉은색 에러 메시지가 도배되지 않습니다.
- 데이터 업데이트 중에 발생하던 일시적인 500 에러(서버 오류)가 사라지고, 사용자에게는 부드러운 로딩 상태가 표시됩니다.
