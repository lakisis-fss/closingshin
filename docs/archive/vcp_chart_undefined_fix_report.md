# 🛠️ 차트 파일명 'undefined_' 에러 해결 보고서

터미널에서 쏟아지던 `undefined_...` 차트 파일 찾기 에러(404)를 해결했습니다.

## 🔍 원인 분석
- **어긋난 이름표**: DB(PocketBase)에는 시장 정보가 `market_name`이라는 이름으로 들어있는데, 화면 코드(`VcpDashboard`, `VcpPageClient`)는 `market`이라는 이름만 찾고 있었습니다.
- **결과**: 시장 정보를 찾지 못해 파일명이 `undefined_종목명_티커.png`로 조립되었고, 실제 파일명(`KOSPI_...` 또는 `KOSDAQ_...`)과 어긋나 에러가 발생했습니다.

## 🛠️ 해결 조치

1. **데이터 매핑 보강 (`api.ts`)**
   - DB에서 데이터를 가져올 때 `market_name`이 있으면 자동으로 `market`으로 옮겨주는 변환기를 설치했습니다.
   - `market: item.market || item.market_name || "UNKNOWN"`

2. **사령탑 코드 수정 (`VcpPageClient.tsx`)**
   - 화면에 주식 목록을 뿌려줄 때 시장 정보가 유실되지 않도록 타입을 보강하여 정확한 값을 전달하게 했습니다.

3. **대시보드 방어벽 설치 (`VcpDashboard.tsx`)**
   - 혹시나 정보가 부족한 종목이 있더라도 에러가 나지 않도록 `UNKNOWN` 처리를 하고, 정보가 아예 없는 경우 "Stock Not Found" 메시지를 보여주도록 튼튼하게 고쳤습니다.

## ✅ 최종 확인 결과
- `대주전자재료 (078600)` 종목을 테스트한 결과, 이제 **`market: KOSDAQ`** 정보를 완벽하게 인식합니다.
- 이에 따라 차트 파일도 `KOSDAQ_대주전자재료_078600.png`라는 올바른 이름으로 찾아내어 화면에 시원하게 뿌려줍니다.

이제 터미널 에러 없이 쾌적하게 차트를 분석하실 수 있습니다! 🏠📉✨
