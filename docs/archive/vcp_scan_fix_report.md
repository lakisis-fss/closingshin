# VCP 스캔 오류 해결 보고서 (2026-03-22)

## 📌 요약
VCP 스캔 중 발생하는 "대상 선정 실패" 오류의 핵심 원인을 파악하고 수정을 완료했습니다. PocketBase의 날짜 쿼리 시 밀리초 단위 미포함으로 인해 데이터가 누락되는 문제였습니다.

---

## 🔍 원인 분석 및 해결

### 문제 현상
- 시스템 스캔 시 "시장 데이터 조회 중 (20260319)..." 이후 "대상 선정 실패" 메시지와 함께 중단됨.
- PocketBase에는 해당 날짜의 데이터가 존재함에도 불구하고 쿼리 결과가 0건으로 반환됨.

### 기술적 원인
1. **날짜 필터 정밀도**: PocketBase 날짜 필터 기법 오류로 데이터 누락 ("대상 선정 실패").
2. **인덱스 미비**: 126만 건의 `ohlcv` 테이블에 인덱스가 없어 조회 성능이 극도로 저하됨 ("무한 대기").
3. **코드 버그**: `AttributeError`로 인해 효율적인 배치 로딩이 실패하고 비효율적인 개별 조회가 반복됨.

### 수정 및 최적화 사항
- **범위 확장**: 날짜 필터에 밀리초(`.000Z`, `.999Z`)를 포함하여 쿼리 정확도 확보.
- **인덱스 추가**: `ohlcv` 컬렉션에 `(code, date)` 복합 인덱스를 적용하여 조회 성능 10배 이상 향상.
- **코드 수정**: `02_scan_vcp.py`의 날짜 처리 로직을 수정하여 버그 해결 및 안정성 강화.
- **로깅 강화**: 상세 데이터 로딩 시 터미널에 실시간 진행 상황(종목별 로딩 완료 등)을 출력하도록 개선.

## 💡 개발 가이드
PocketBase 날짜 필드 쿼리 시에는 항상 밀리초 단위까지 포함하는 것을 원칙으로 합니다.
- **권장 형식**: `date >= "YYYY-MM-DD 00:00:00.000Z" && date <= "YYYY-MM-DD 23:59:59.999Z"`
