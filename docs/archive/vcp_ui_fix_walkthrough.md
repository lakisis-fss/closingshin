# VCP 프론트엔드 UI 및 인증 시스템 개선 보고서 (Walkthrough)

## 1. 작업 개요
VCP 스캔 관련 인증 에러 해결 및 프론트엔드 달력 표시 오류 등 사용자 경험(UX) 개선 작업을 완료했습니다.

## 2. 주요 조치 사항
### 인증 시스템 안정화 (ScriptRunner)
- **대상 파일**: `frontend/backend/scriptRunner.ts`
- **문제**: 로그 생성 시 동시 인증 요청이 발생하여 PocketBase SDK가 이를 자동 취소(Abort)하던 에러.
- **조치**: 
    - `pb.autoCancellation(false)` 설정으로 서버 측 요청 안정성 확보.
    - **싱글톤 Promise 패턴** 적용: 인증이 진행 중일 때 추가 요청을 보내지 않고 기존 요청을 대기하여 중복 인증 방지.

### 달력 스캔 현황 표시 오류 해결
- **대상 파일**: `frontend/src/lib/api.ts`
- **문제**: `limit: 500` 설정으로 인해 과거 데이터(3월 23일 등)가 조회 목록에서 누락되어 달력에 밑줄(Scanned)이 표시되지 않음.
- **조치**: 스캔 날짜 조회 한도를 `5000`으로 상향하여 누락 없는 상태 표시 보장.

### 시스템 최종 스캔 시간 오류 수정
- **대상 파일**: `frontend/src/lib/api.ts`
- **문제**: 로컬 시간 문자열 변환 방식 차이로 인해 화면에 `INVALID DATE`로 표시됨.
- **조치**: 백엔드에서 ISO 표준 문자열을 반환하도록 수정하여 브라우저에서 정확한 시간 변환이 가능하도록 개선.

## 3. 검증 결과
- **인증 에러**: `npm run scheduler` 실행 시 더 이상 `ClientResponseError 0` 에러가 발생하지 않음.
- **달력 표시**: 3월 23일 데이터를 포함하여 3월 전체 스캔 현황이 정상적으로 표시됨을 DB 쿼리로 교차 검증 완료.
- **시간 표시**: `SYS.LAST` 항목이 KST 기준으로 정확히 표시됨.

상세 분석 내용은 `/docs/pb_auth_error_analysis.md`를 참고하세요.
