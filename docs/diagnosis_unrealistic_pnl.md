# 포트폴리오 수익률 오기입 진단 및 조치 보고서

이 문서는 기가비스(420770) 종목에서 발생한 비현실적인 수익률(+931%)의 원인 진단과 조치 결과를 정리한 문서입니다.

## 1. 문제 현상
- **종목**: 기가비스 (420770)
- **현상**: 수익률이 **+931.05%**로 표시됨 (2일 만에 약 10배 상승)
- **원인 데이터**: 청산가(Exit Price)가 **797,000원**으로 기입됨

## 2. 진단 결과
- **원인**: 수동 청산(Manual Close) 시 가격 입력 과정에서 **0이 하나 더 입력된 오타**로 판명되었습니다.
    - 실제 의도: 79,700원 (당일 종가 78,700원과 인접)
    - 입력된 값: 797,000원
- **발생 시점**: 당일 고가/저가 범위를 체크하는 유효성 검사 로직이 도입되기 이전에 입력된 데이터로 확인되었습니다.

## 3. 조치 내용
- **1차 시도 실패**: `simulation_data`라는 잘못된 필드명으로 업데이트를 시도하여 실제 DB에 반영되지 않았습니다.
- **최종 조치**: PocketBase SDK를 직접 사용하여 `simulation` 필드 내의 `{ "exitPrice": 79700, "realizedPnlPercent": 3.10 }` 데이터를 정상적으로 업데이트 완료하였습니다.
- **데이터 수정**: 기가비스 레코드(`fqktbgog6xy4089`)의 청산 데이터를 아래와 같이 보정하였습니다.
    - **변경 전**: 797,000원 (+931.05%)
    - **변경 후**: **79,700원 (+3.10%)**
- **PnL 재계산**: 79,700원을 기준으로 실현 손익(Realized PnL)을 72,000원(+3.10%)으로 업데이트 완료하였습니다.

## 4. 예방 조치 (현재 적용됨)
- 현재 시스템에는 `ClosePositionModal.tsx`를 통해 청산일의 **고가(High)와 저가(Low)를 조회**하여, 입력값이 범위를 벗어날 경우 저장이 불가능하도록 하는 유효성 검사 로직이 이미 적용되어 있습니다.
- 향후 이러한 오타로 인한 수익률 왜곡은 발생하지 않을 것입니다.
