# 백테스트(Backtest) 페이지 아이디어 제안

**Total Quant Score**를 기반으로 종목을 선정하고, 다양한 매매 시나리오(진입/청산)를 적용했을 때의 성과를 시뮬레이션하는 기능입니다.

---

## 🧠 Brainstorm: Backtest Simulation Strategy

### Context
사용자는 **Total Quant Score**라는 강력한 지표를 가지고 있습니다. 이 점수를 신뢰하여 상위 종목을 매수했을 때, 과연 수익이 나는지, 그리고 어떤 청산 전략(익절/손절/트레일링 등)이 가장 효과적인지 검증하고 싶어 합니다.

---

### Option A: "History Replay" (과거 시점 조회)

**"내가 저번 달 1등 종목을 샀다면 지금 얼마일까?"**
가장 직관적이고 빠르게 구현 가능한 모델입니다.

-   **작동 방식**:
    1.  사용자가 **과거 날짜**(예: 2024년 1월 1일)를 선택.
    2.  시스템이 **그 날짜 기준** Total Quant Score 랭킹을 로드.
    3.  사용자가 상위 N개 종목(또는 특정 종목)을 선택.
    4.  **현재 시점(Today)**까지 보유했을 때의 수익률을 계산.
-   **청산 조건**: 단순 보유(Buy & Hold) 또는 목표가 도달 시 확정 수익 계산.

✅ **Pros:**
-   구현 난이도가 낮음 (복잡한 시뮬레이션 로직 불필요).
-   결과가 매우 직관적임.

❌ **Cons:**
-   일별 트레이딩(매수/매도 반복) 시뮬레이션은 어려움.
-   "타임 컷"이나 "트레일링 스탑" 같은 동적인 대응 로직 적용이 제한적.

📊 **Effort:** Low

---

### Option B: "Strategy Lab" (정밀 시뮬레이터)

**"전략을 세우고, 기계처럼 매매했다면 결과는?"**
사용자가 정의한 **Rule**에 따라 시스템이 일별 데이터를 순회하며 매매를 수행합니다.

-   **작동 방식**:
    1.  **기간 설정**: 시작일 ~ 종료일.
    2.  **유니버스 설정**: "Total Quant Score 상위 5개 종목 매수".
    3.  **매매 규칙(Exit Rules) 설정** (상세 제어 가능):
        -   **익절(Take Profit)**: +10% 도달 시 매도.
        -   **손절(Stop Loss)**: -5% 도달 시 매도.
        -   **타임 컷(Time Cut)**: 5일 보유 후 무조건 매도.
        -   **트레일링 스탑(Trailing Stop)**: 고점 대비 -3% 하락 시 매도.
    4.  시스템이 날짜별로 진행하며 매수/매도 이벤트를 발생시키고 계좌 잔고 변화를 기록.

✅ **Pros:**
-   실전과 가장 유사한 데이터 제공 (MDD, 승률, 손익비 등).
-   다양한 전략의 우위를 비교 분석 가능.

❌ **Cons:**
-   구현 난이도가 높음 (일별 OHLCV 데이터 기반 이벤트 처리 로직 필요).
-   계산 시간이 걸릴 수 있음.

📊 **Effort:** High

---

### Option C: "Interactive Playground" (실시간 슬라이더) 

**"수치를 조절하며 황금 비율 찾기"**
Option B의 경량화 버전으로, UI/UX에 초점을 맞춥니다.

-   **작동 방식**:
    1.  좌측에 슬라이더 컨트롤 패널 배치 (익절율, 손절율, 보유기간 등).
    2.  우측에 수익률 그래프와 핵심 지표(Total Return, MDD) 표시.
    3.  사용자가 슬라이더를 드래그하면 **실시간**으로 차트가 변함.
    4.  무거운 전체 시뮬레이션 대신, 미리 계산된 데이터나 근사치를 활용하여 반응 속도를 극대화.

✅ **Pros:**
-   사용자 경험(UX)이 압도적으로 좋음. (게임하듯이 전략 최적화 가능)
-   "손절을 3%로 줄이면 수익이 어떻게 변하지?" 같은 질문에 즉답 가능.

❌ **Cons:**
-   실시간성을 위해 데이터 구조 설계가 까다로움.

📊 **Effort:** Medium ~ High

---

## 💡 Recommendation

**Option B (Strategy Lab) + Option C (UI 요소 일부 차용)** 를 추천합니다.

1.  백테스트의 핵심은 **"신뢰도"**이므로, 정밀한 시뮬레이션(Option B)이 기본이 되어야 합니다.
2.  하지만 설정이 너무 복잡하면 사용하기 어려우므로, 입력 폼을 Option C처럼 직관적인 UI(슬라이더, 토글)로 구성하여 편의성을 높이는 방향이 좋습니다.

### 🚀 구현 로드맵 (Proposed Roadmap)

1.  **데이터 준비**: 일별 OHLCV 데이터 및 과거 Total Quant Score 데이터 확보.
2.  **Backtest Engine (Python)**:
    -   입력: 종목 리스트, 기간, 청산 규칙.
    -   출력: 일별 자산 변화(Equity Curve), 거래 내역(Trade Log), 통계(MDD, CAGR).
3.  **Frontend**:
    -   **설정 패널**: 기간 선택, 자산 배분 설정, 청산 조건(익절/손절/TS) 입력.
    -   **결과 리포트 Dashboard**:
        -   수익률 그래프 (벤치마크 vs 내 전략).
        -   핵심 지표 카드 (총 수익, MDD, 승률, 평균 손익비).
        -   월별 수익률 히트맵 (Monthly Heatmap).
