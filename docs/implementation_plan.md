# 🚀 ClosingSHIN 성능 최적화 구현 계획서

사용자님께서 보고하신 **"메뉴 이동 시 반응 없음 및 페이지 전환 지연"** 문제를 해결하기 위한 기술적 분석과 해결 방향을 정리했습니다.

---

## 🔎 현재 원인 분석 (Diagnostic Results)

현재 시스템에서 발견된 병목 현상의 주범은 **'순차적 데이터 호출(Sequential Waterfall)'**과 **'비효율적인 DB 쿼리'**입니다.

1. **Next.js 서버 컴포넌트 지연**: 대시보드 및 VCP 페이지 이동 시 4~8개의 데이터를 비효율적인 순서로 불러오고 있습니다.
2. **PocketBase 과부하**: `getAvailableVcpDates` 함수가 매번 5,000건의 데이터를 조회하면서 시스템 전체 응답 속도를 늦추고 있습니다.
3. **사용자 피드백 부재**: 로딩 중임을 알려주는 UI 요소가 없어 사용자가 링크 클릭 시 "작동하지 않는다"고 느낄 수밖에 없는 구조입니다.

---

## 🛠️ 해결 제안 (Proposed Solutions)

### 1. 데이터 호출 병렬화 (Parallel Execution)
- `await`를 순차적으로 쓰지 않고 `Promise.all`로 묶어 동시에 실행합니다.
- 예시: 5개가 순차적으로 0.5초씩 총 2.5초 걸리던 페이지가, 병렬 처리 시 **0.5초** 만에 로딩됩니다.

### 2. 쿼리 최적화 (Query Optimization)
- 데이터베이스 호출 시 꼭 필요한 **필드(Fields)**만 선택해서 가져와 전송 효율을 높입니다.
- 불필요하게 많이 가져오던 `Limit` 설정을 현실적으로(100개 이하 등) 조정합니다.

### 3. 로딩 시각화 (Loading UI)
- 사용자 규칙에 맞춰 `loading.tsx`를 도입하여, 페이지가 준비되는 동안 멋진 **바우하우스 스타일 로딩 디자인**을 보여줍니다.
